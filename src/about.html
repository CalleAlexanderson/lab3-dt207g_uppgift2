Webbsidans funktion

Webbsidan visar information om de tidigare jobb jag jobbat på. Informationen om jobben är lagrade i en databas som ligger i ett API som sedan anropas med hjälp av javascript.

API:et använder sig av CRUD och har funktion för GET, POST och DELETE, när ett fetch anrop görs med GET (och korrekt url) så skickas det tillbaks json data med de olika raderna i databasens tabbel som objekt.

POST lägger till en ny arbetspost i tabellen och anropas med body parametrar som tas från ett formulär på add.html.

DELETE tar bort den rad från tabellen som man klickat på, detta görs genom att ett ID skickas med som en parameter i URL som används vid API anropet.

Mysql

Jag har använt MySql(MariaDB) för att skapa min databas, databasen skapade jag lokalt på min dator genom kommandtolken och tabellen och de rader som läggs in skapas med hjälp av javascript.

Slutsatser

Jag tyckte detta sätt att lagra data på en databas och sedan göra ett API till den var lättare både att skapa och sedan också lättare att jobba med än det sätt vi gjorde i moment 1 då jag redan lärde mig hur man gjorde anrop mot API i tidigare kurs och det var lättare att hålla koll på min kod då det låg uppdelat i olika projekt för både front- och backend.
