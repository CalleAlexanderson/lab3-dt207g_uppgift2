Webbsidans funktion

Webbsidan visar information om de tidigare jobb jag jobbat på. Informationen om jobben är lagrade i en databas som ligger i ett API som sedan anropas med hjälp av javascript.

API:et använder sig av CRUD och har funktion för GET, POST och DELETE, när ett fetch anrop görs med GET (och korrekt url) så skickas det tillbaks json data med de olika dokumenten i databasens collection som objekt.

POST lägger till en ny arbetspost i collection och anropas med body parametrar som tas från ett formulär på add.html.

DELETE tar bort det dokument från collection som man klickat på, detta görs genom att ett ID skickas med som en parameter i URL som används vid API anropet.

MongodDB

Jag har använt MongodDB för att skapa min databas, databasen skapade jag lokalt på min dator genom Compass och collection samt all data skapas med javascript.

Slutsatser

Jag tycker mer om detta system att hantera data i en database mer än MySql då jag tyckte det här var smidigare och jag gillar att kunna lägga in vad jag än vill i min databas men även kunna göra ett schema om bara specifik data ska användas.
